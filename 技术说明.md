# 技术说明

## 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | Vue 3 + Vite + Pinia + TypeScript | 三栏式布局，WebSocket 状态驱动 |
| 后端 | Go 1.25 + sqlc + Goose | SQLite (WAL mode)，单二进制部署 |
| 通信 | WebSocket (JSON/Proto 双模式) | 开发环境用 JSON（`WS_CODEC=json`），生产可用 Proto |
| 配置 | `actions.json` + `attributes.json` | 嵌入二进制，启动时加载校验 |
| E2E 测试 | Playwright | Chromium，单 worker（SQLite 串行要求）|

## 核心数据文件

| 文件/目录 | 说明 |
|-----------|------|
| `docs/actions.json` | 游戏核心数据配置，定义所有物品和事件的元数据 |
| `docs/actions_format.md` | `actions.json` 的格式说明文档 |
| `docs/attributes.json` | 属性定义及静态修饰器模板 |
| `docs/游戏文档/` | 各系统的详细设计文档 |

`actions.json` 和 `attributes.json` 均通过 `//go:embed` 嵌入二进制，启动时加载并校验。变更后需运行 `go run ./cmd/genregistry` 更新数字 ID 映射（`id_registry.json`、`attr_registry.json`）。

## 后端架构概览

### 通信模型

- **全局固定间隔 Tick**：`Manager.TickAll` 启动 `time.Ticker`（默认 50ms），每个 Tick Round 并行处理所有在线 Session。
- **双阶段 runTick**：Phase 1 排空命令，Phase 2 事件结算。增量更新 `state.diff` 在 runTick 内部立即推送。
- **批量刷盘**：Tick Round 结束后，`BatchFlush` 将所有脏 Session 在一个 SQLite 事务中批量持久化。
- **无应用层心跳**：服务端不依赖应用层心跳判定连接存活，WebSocket `onclose` 触发 Grace Timer（默认 30s）。底层 `wsx.Conn` 会按 `PingInterval`（默认 30s）发送 WebSocket ping 帧保活。
- **单在线**：同一 `userID` 只能有一个活跃 WS 连接。新连接复用内存中的 `PlayerSession`。

### 数据库表

| 表 | 用途 |
|----|------|
| `users` | 玩家账号 |
| `sessions` | HTTP Session（cookie 认证） |
| `player_init` | 新玩家初始化标记 |
| `player_skills` | 技能等级和经验 |
| `player_inventory` | 库存（`quantity` 为 REAL，支持小数） |
| `player_equipment` | 装备槽位 |
| `player_unlocked_events` | 已解锁事件（图鉴） |
| `player_discovered_items` | 已发现物品（图鉴） |
| `player_active_events` | 事件队列 |

### 关键设计

- **属性系统**：懒计算 + 脏标记传播，三阶段计算（ADD → MULTIPLY → OVERRIDE）。
- **记录系统**：`record.Recorder`（命名空间栈）→ `Registry.BuildDiff`（跨命名空间合并）→ `pb.StateDiff`。
- **图鉴持久化**：`player_discovered_items` 表存储物品发现记录，`bestiary.State` 有 `dirtyItems` 增量刷盘。

## 前端架构概览

### 状态管理

Pinia `useGameStore` 管理游戏状态：
- `fetchStaticData()` — 从 `/api/v1/game/config` 加载静态配置
- `initWsListeners()` — 注册 `state.full` / `state.diff` / `error` 处理器
- `applyStateFull(full)` — 完整状态初始化
- `applyStateDiff(diff)` — 增量更新合并

### WS 连接管理

`lib/ws.ts` 提供全局 WS 管理：
- `connect()` / `disconnect()` — 建立/优雅关闭连接
- `send()` / `sendAndWait()` — 发送消息（自动关联请求/响应）
- `onMessage()` — 注册推送消息处理器
- 自动重连机制（指数退避），但 `disconnect()` 设置 `intentionalClose=true` 阻止重连

`MainView.vue` 卸载时必须调用 `disconnect()`，防止页面离开后 WS 在后台无限重连。

## 开发命令

```bash
# 后端
cd backend
go run ./cmd/server        # 启动服务端 (:8080)

# 前端
cd web-client
npm run dev                # 启动开发服务器 (:5173)
npm run test:e2e           # 运行 E2E 测试

# 配置变更后重新生成 ID 映射
cd backend
go run ./cmd/genregistry
```

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `WS_CODEC` | `json` | WebSocket 编码：`json` 或 `proto` |
| `WS_SESSION_GRACE_PERIOD` | `30s` | Session 宽限期 |
| `WS_GAME_LOOP_TICK` | `50ms` | Tick 间隔 |
| `HTTP_ADDR` | `:8080` | HTTP 监听地址 |
