# 区域系统

区域（Area/Map）用于对事件等内容进行分类和组织，对应 `actions.json` 中的 `map` 字段。

## 设计目的

1. **内容组织**：将事件按地理位置或功能场景分组，便于前端展示和玩家理解。
2. **渐进解锁**：新区域通过剧情事件（`upgrade` 类型）逐步开放，形成游戏进度节奏。
3. **区域特性**：未来可为不同区域设置环境加成、专属资源或特殊机制。

## 当前区域

| 地图 ID | 名称 | 说明 |
|---------|------|------|
| `village` | 村庄 | 游戏初始区域，包含基础采集、制造和主线剧情事件 |

当前所有事件均归属 `village`。随着游戏内容扩展，可新增 `forest`、`mine`、`dungeon` 等区域。

## 区域与事件的关系

每个事件在 `actions.json` 中必须指定一个 `map` 字段。后端提供以下索引：

- `EventsByMap(mapID)`：获取指定地图下的所有事件。
- `EventsByMapID(mapID)`：同上，接受数字 ID。

## 数据存储

区域使用稳定的数字 ID（`MapID`，`int64`）存储，当前定义在 `id_registry.json` 中。

玩家解锁的区域目前不单独存储（所有玩家默认在 `village`）。未来若需区域解锁机制，可新增 `player_unlocked_maps` 表：

```sql
CREATE TABLE player_unlocked_maps (
    user_id   INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    map_id    INTEGER NOT NULL,
    unlocked_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, map_id)
);
```

## 扩展方向

- 多区域并行：玩家可同时向不同区域派遣事件队列。
- 区域 Buff/Debuff：特定区域内的事件效率受环境因素影响。
- 区域专属资源：某些物品仅在特定区域的事件中产出。
