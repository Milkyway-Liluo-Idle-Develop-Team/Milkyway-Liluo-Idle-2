# 进度总结机制

进度总结用于向玩家展示阶段性成果，增强成就感和游戏黏性。

## 触发时机

进度总结可在以下时机触发：

1. **定时总结**：玩家在线达到一定时长（如 30 分钟、1 小时）后自动弹出。
2. **手动查看**：玩家通过 UI 主动请求查看某个时间段的总结。
3. **离线回归**：玩家离线一段时间后重新登录，展示离线期间的产出概要。
4. **里程碑**：完成重要剧情、解锁新区域、技能升级等关键节点。

## 总结内容

### 基础统计

| 统计项 | 说明 |
|--------|------|
| 在线时长 | 本次游戏会话或总结周期内的在线时间 |
| 事件循环次数 | 各事件完成的循环总数 |
| 资源产出 | 获得的各种物品数量汇总 |
| 经验获取 | 各技能获得的经验值 |
| 技能升级 | 本次周期内提升等级的技能列表 |

### 效率分析

- **最高效事件**：单位时间内产出价值最高的事件。
- **资源转化率**：消耗的材料 vs 产出的价值。
- **经验/时间比**：各技能的经验获取效率排名。

### 成就提示

- 首次获得某种稀有物品。
- 技能达到特定等级（如 crafting 达到 10 级）。
- 累计完成特定事件达到一定次数。

## 数据来源

进度总结的数据直接从玩家状态表和**行为日志**中聚合：

- 实时状态：`player_inventory`、`player_skills`、`player_active_events`。
- 历史记录：行为日志表中按时间范围聚合。

## 实现方式

后端提供专用的查询接口，按时间范围聚合日志数据：

```sql
-- 示例：统计某时间段内的物品产出
SELECT item_id, SUM(quantity_delta) as total
FROM behavior_logs
WHERE user_id = ?
  AND type = 'item_gain'
  AND created_at BETWEEN ? AND ?
GROUP BY item_id;
```

前端根据返回的统计数据渲染可视化图表（柱状图、饼图、时间线等）。

## 扩展方向

- **对比功能**：与上一周期对比成长幅度。
- **目标追踪**：玩家设定短期目标（如"获得 100 个 oak_logs"），总结时显示完成进度。
- **分享功能**：生成进度报告卡片，支持截图分享。
