# 其他系统或机制

本章收录尚未独立成章的辅助系统和游戏机制。

## 战斗系统

战斗系统目前通过装备（`equipment`）和战斗技能（`battle_skills`）实现基础框架。

### 战斗技能 (BattleSkill)

装备在 `equipment_details.battle_skills` 中定义附带的战斗技能：

| 字段 | 说明 |
|------|------|
| `id` | 技能唯一标识 |
| `name` | 显示名称 |
| `description` | 描述文本 |
| `target_type` | 目标类型，如 `"single"`（单体） |
| `damage` | 伤害配置对象 |
| `is_basic` | 是否为普攻 |

### 伤害配置 (DamageProfile)

| 字段 | 说明 |
|------|------|
| `type` | 伤害类型，如 `"physical"` |
| `flat` | 固定伤害值 |
| `multiplier` | 伤害倍率 |

最终伤害计算公式（暂定）：

```
damage = flat + (角色属性 × multiplier)
```

### 战斗经验

战斗相关经验归属 `strength` 技能，通过击败敌人或使用战斗技能获得。

## 工具系统

工具（`tool == true` 的物品）装备后可为特定类型的事件提供效率加成。

### 工具位置 (ToolPosition)

`tool_position_requirement` 定义工具生效的位置要求：

```json
{
  "tool_position": "main_hand",
  "value": 1
}
```

### 工具加成

`tool_basic_data` 中的键值对用于事件结算时的产出计算。例如：

- `"axe"` 类型工具可能提升伐木事件的产出数量。
- `"pickaxe"` 类型工具可能缩短采矿事件的 `loop_time`。

具体加成公式由事件结算引擎实现。

## 装备强化与锻造

### 强化 (Enhance)

使用 `enhancing` 技能对装备进行强化，提升 `equipment_upgrade_data` 中的属性数值。强化成功率由 `upgrade_curve` 中的 `basic_success_rate` 决定。

### 锻造 (Forge)

使用 `forging` 技能在装备的 `forge_slot` 中镶嵌特殊效果。锻造可能改变装备的 `element` 或新增额外属性。

## 剧情系统

剧情通过一系列 `upgrade` 类型事件推进：

- `main_story_*`：主线剧情，解锁核心游戏内容。
- `starting_dialog_*`：新手引导对话。
- `xgs_sidestory_*`：支线剧情，提供额外背景故事和奖励。

完成剧情事件会将其加入 `player_unlocked_events`，并可能开放新的可执行事件。

## 存档与同步

- 所有玩家数据存储在服务端 SQLite 数据库中。
- 客户端通过 WebSocket 维持长连接，定期发送同步请求获取最新状态。
- 离线期间的事件产出在下次同步时一次性结算。
- 玩家切换设备时，只需重新登录即可恢复完全一致的游戏进度。
