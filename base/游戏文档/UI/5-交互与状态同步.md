# 交互与状态同步

## 增量更新处理优先级

收到服务端增量更新包后，前端按以下顺序处理：

1. **库存变化** (`inventory_changes`)：
   - 立即更新右侧库存面板对应物品数量。
   - 播放数值缩放动画，若数量增加则飘出 `+N` 浮动提示。
   - 新物品以渐入动画加入列表。

2. **事件执行** (`event_executions`)：
   - 更新顶部栏进度条。
   - 更新事件卡片上的 `"触发次数"` 累计值。

3. **队列更新** (`queue_updates`)：
   - `scope == "current"`：仅刷新顶部栏和队列下拉中当前事件的 `progress`。
   - `scope == "full"`：全量刷新队列列表（事件移除、reorder 等结构性变化）。

4. **技能经验** (`skill_xp_changes`)：
   - 更新左侧属性面板对应技能的等级数值。
   - 若触发升级 (`old_level != new_level`)，播放升级特效（如数值放大、金色闪烁）。

5. **属性变化** (`attribute_changes`)：
   - 若左侧战斗属性面板处于展开状态，实时更新对应属性值。
   - 属性值变化时播放过渡动画。

6. **修饰器变化** (`modifier_changes`)：
   - 主要用于调试和来源追踪，通常不直接展示在 UI 上。

7. **图鉴解锁** (`bestiary_unlocks`)：
   - 在对应分类（事件/物品/区域）上显示红点提示 `"新!"`。

## 断线重连处理

1. WebSocket 断开时，显示 `"连接中..."` 遮罩，不阻塞本地展示。
2. 重连成功后，请求完整状态快照。
3. 用新快照全量刷新所有面板，平滑替换当前状态（避免数值突兀跳变，可用短暂过渡动画）。

## 操作反馈

| 操作 | 反馈 |
|------|------|
| 点击 `"+队列"` | 按钮短暂缩放动画，若队列面板可见则新项滑入 |
| 点击 `"开始循环"` | 按钮文案变为 `"循环中"`，主按钮样式变为激活态（如蓝色）|
| 点击 `"停止"` | 按钮文案恢复，样式恢复默认 |
| 修改目标次数 | 输入框失焦后发送请求，成功无提示，失败toast提示 |
| 穿戴装备 | 装备卡片出现 `"[已穿戴]"` 标签，属性面板数值刷新 |
