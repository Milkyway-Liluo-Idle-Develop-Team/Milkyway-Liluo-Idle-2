# 中间场景面板

中间主区域展示当前场景和可执行的事件列表，是玩家编排生产活动的核心操作区。

## 布局

```
┌─────────────────────────────────────────────┐
│  场景                                        │
│  ┌─────────┐                                │
│  │  村庄   │                                │
│  └─────────┘                                │
├─────────────────────────────────────────────┤
│  循环行动                                    │
│  ┌───────────────────────────────────────┐  │
│  │ 砍伐橡木                    [可执行]  │  │
│  │ 要致富，先撸树——不对，这里不是银河    │  │
│  │ MC放置。                               │  │
│  │                                         │  │
│  │ [耗时 2.00s] [产出 橡木原木 1->1.25]  │  │
│  │ [经验 20] [触发次数 72015]             │  │
│  │                                         │  │
│  │ [砍伐 大于等于 1]                     │  │
│  │                                         │  │
│  │ [循环中（点击停止）] [∞] [+队列]      │  │
│  └───────────────────────────────────────┘  │
│  ...                                        │
└─────────────────────────────────────────────┘
```

## 场景切换

- 顶部横向排列已解锁区域（如 `"村庄"`），当前场景高亮显示。
- 未解锁区域显示为锁定状态，点击弹出解锁条件提示。
- 切换场景时，下方事件列表平滑过渡刷新。

## 事件卡片

每张事件卡片为圆角矩形卡片，包含以下元素：

| 元素 | 说明 |
|------|------|
| 事件名称 | 左侧大字号标题，如 `"砍伐橡木"` |
| 状态标签 | 右侧 `"[可执行]"` 标签，若 requirements 未满足则显示 `"[条件不足]"` 并置灰 |
| 描述文本 | 事件描述，如 `"要致富，先撸树——不对，这里不是银河MC放置。"` |
| 信息标签行 | 横向排列的灰色小标签，展示关键信息 |
| 需求标签 | 执行所需的 skill/item/fluid/event 条件，如 `"[砍伐 大于等于 1]"` |
| 操作区 | 主按钮 + 目标次数设置 + 加入队列按钮 |

### 信息标签

| 标签 | 示例 | 说明 |
|------|------|------|
| 耗时 | `[耗时 2.00s]` | loop 事件的 `loop_time` |
| 产出 | `[产出 橡木原木 1->1.25]` | 单次产出数量，含小数表示加成后数值 |
| 经验 | `[经验 20]` | 单次获得经验值 |
| 触发次数 | `[触发次数 72015]` | 该事件累计执行次数（历史统计）|

### 操作区

| 元素 | 说明 |
|------|------|
| 主按钮 | `"开始循环"` / `"循环中（点击停止）"`，蓝色激活态表示正在执行 |
| 目标次数 | 输入框或下拉，默认 `"∞"`，可设置有限次数 |
| 加入队列 | `"+队列"` 按钮，点击将事件加入主队列末尾 |

## 事件状态

### 可执行

- 所有 requirements 满足，按钮可点击。
- 主按钮显示 `"开始循环"` 或 `"循环中"`。

### 条件不足

- 某项 requirement 未满足（如技能等级不够、缺少前置事件）。
- 卡片整体 opacity 降低，状态标签显示 `"[条件不足]"`。
- 主按钮置灰不可点击，显示具体缺失条件。

### 已在队列

- 若该事件已在当前队列中，显示 `"已在队列"` 提示，避免重复添加。

## 事件可见性

- 默认展示当前场景下所有事件（含未解锁的）。
- 未解锁事件置灰显示，并提示解锁方式（引导探索）。
- 完全隐藏的事件（如属于未解锁区域的）不在当前场景显示。

## 实时更新

- 事件执行时， `"触发次数"` 标签实时累加。
- 若当前事件产出发生变化（如装备加成变动）， `"产出"` 标签数值平滑更新。
