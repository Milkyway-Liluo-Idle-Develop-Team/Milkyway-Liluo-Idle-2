# 行为日志

游戏的所有产生**状态变化**的行为都会生成一条日志。日志用于回放玩家操作轨迹、排查异常、以及支撑部分游戏功能（如进度总结）。

## 记录范围

以下行为会触发日志记录：

- **事件结算**：每次事件循环完成后，记录实际产出和消耗。
- **物品变动**：获得、消耗、丢弃物品。
- **技能升级**：经验增加、等级提升。
- **装备操作**：穿戴、卸下、强化、锻造装备。
- **解锁事件**：完成 `upgrade` 类型事件后解锁新内容。
- **玩家操作**：手动调整队列、切换地图等。

## 日志结构

每条日志至少包含：

| 字段 | 说明 |
|------|------|
| 时间戳 | 行为发生的精确时间 |
| 玩家 ID | 行为归属玩家 |
| 行为类型 | 如 `event_loop`, `item_gain`, `skill_levelup` |
| 关联对象 | 如事件 ID、物品 ID、技能 ID |
| 变动数值 | 数量、经验值等 delta 值 |
| 上下文 | 当时的队列状态、属性加成系数等 |

## 用途

1. **数据审计**：追溯玩家资源来源，排查异常产出。
2. **进度总结**：基于日志聚合生成玩家的阶段性成果报告。
3. **离线补偿校验**：离线结算结果可通过日志逐项核对。

## 日志存储

日志只保留48h，不存储在主数据库内
