# 属性系统

属性系统定义了角色和装备的能力数值，以及这些数值如何被事件、工具和装备所修饰。

## 属性定义

属性以键值对形式存储在装备或工具的详情对象中，没有固定的全局属性列表。常见的属性包括：

| 属性键 | 含义 | 典型来源 |
|--------|------|----------|
| `attack_interval` | 攻击间隔（秒） | 武器基础数据 |
| `physical_power` | 物理攻击力 | 武器基础/升级数据 |
| `accuracy` | 命中率 | 武器基础/升级数据 |
| `hatred_multiplier` | 仇恨倍率 | 防具升级数据 |

## 属性来源

### 1. 装备基础属性 (`equipment_basic_data`)

装备在 `equipment_details.equipment_basic_data` 中定义基础数值。例如：

```json
"equipment_basic_data": {
  "attack_interval": 2,
  "physical_power": 10,
  "accuracy": 10
}
```

### 2. 装备升级属性 (`equipment_upgrade_data`)

装备每次升级后，属性会按 `upgrade_curve` 中的倍率放大。升级后的数值存储在 `equipment_upgrade_data` 中：

```json
"equipment_upgrade_data": {
  "physical_power": 10,
  "accuracy": 10
}
```

### 3. 工具加成属性 (`tool_basic_data`)

工具对事件产出的加成系数存储在 `tool_details.tool_basic_data` 中，格式为开放的键值对象。

## 属性修饰机制

1. **基础值**：来自装备/工具的 `*_basic_data`。
2. **升级倍率**：`upgrade_curve` 中的 `ability_multiplier` 用于放大基础值。
3. **状态位**：`player_inventory.item_state` 字段（32 位整数）可用于存储装备的变体描述符，如强化等级、锻造词缀等。不同的 `item_state` 对应同一 `item_id` 的不同属性版本。
4. **运行时计算**：最终属性 = 基础值 × 升级倍率 + 外部加成（技能等级、Buff 等）。

## 战斗属性

战斗相关的属性在 `BattleSkill` 的 `damage` 对象中定义：

| 字段 | 说明 |
|------|------|
| `type` | 伤害类型，如 `"physical"` |
| `flat` | 固定伤害值 |
| `multiplier` | 倍率系数 |

最终伤害由角色属性、装备加成和技能倍率共同计算得出。
