# 经验系统

经验系统负责记录和管理玩家在各技能上的成长进度。

## 技能定义

当前游戏中的技能（`skill`）包括：

| 技能 ID | 名称 | 说明 |
|---------|------|------|
| `crafting` | 制作 | 制作木板、工具、装备等 |
| `enhancing` | 强化 | 装备升级与附魔相关 |
| `felling` | 伐木 | 采集木材类资源 |
| `forging` | 锻造 | 冶炼金属、凝结黑石等 |
| `magic` | 魔法 | 法术相关 |
| `mining` | 采矿 | 采集矿石、泥土、石头等 |
| `strength` | 力量 | 战斗与物理相关 |

每个技能拥有独立的等级（`level`）和经验值（`xp`）。

## 经验获取

经验通过执行事件获得。在 `actions.json` 中，事件的 `rewards` 数组里可以包含 `type` 为 `"experience"` 的奖励：

```json
{
  "type": "experience",
  "value": 20.0,
  "skill_id": "felling"
}
```

- `value`：获得的经验数值（支持小数）。
- `skill_id`：经验归属的目标技能。

事件每次完成循环（`loop` 类型）或执行（`instant` 类型）时，对应的经验奖励会累加到玩家该技能的经验池中。

## 技能升级

当技能经验值达到当前等级上限时，技能等级提升。升级可能带来：

- 解锁更高等级要求的新事件。
- 提升相关事件的效率（如循环时间缩短、产出增加）。
- 满足装备穿戴或工具使用的前置条件。

## 数据存储

玩家技能数据存储在 `player_skills` 表中：

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_id` | INTEGER | 玩家 ID |
| `skill_id` | INTEGER | 技能数字 ID |
| `level` | REAL | 当前等级 |
| `xp` | REAL | 累计经验值 |
| `updated_at` | DATETIME | 最后更新时间 |

主键为 `(user_id, skill_id)`，即每个玩家对每个技能只有一行记录。

## 更新记录

经验系统在技能经验发生变动时向当前命名空间写入记录。

### 技能经验记录

```go
type SkillXpRecord struct {
    SkillID  SkillID
    XpDelta  float64   // 本次获得的经验值（可正，目前无扣除场景）
    NewLevel float64   // 变动后等级
}
```

**产生时机**：事件结算发放经验奖励、其他系统（如成就、Buff）直接给予经验时写入。

### 合并规则

同一 `SkillID` 在**同一命名空间**内可累加 `XpDelta`。`OldLevel` 取最早值，`NewLevel` 取最终值。

### 序列化示例

```json
{
  "skill_xp_changes": [
    {"skill_id": 4, "xp_delta": 120.0, "old_level": 3.0, "new_level": 4.0}
  ]
}
```
