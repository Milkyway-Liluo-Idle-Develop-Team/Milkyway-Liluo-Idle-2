# UI 设计概览

## 整体布局

界面采用**三栏式布局**，顶部固定标题栏，中间分为左中右三个主区域。

```
┌─────────────────────────────────────────────────────────────┐
│  顶部栏 (TopBar)                                             │
│  [图标] 标题  |  当前事件进度  |  队列  |  状态  |  功能按钮   │
├─────────────┬─────────────────────────────┬─────────────────┤
│             │                             │                 │
│  左侧属性    │        中间场景面板          │   右侧库存      │
│  面板        │        (场景/事件详情)       │   面板          │
│             │                             │                 │
│ 生产属性     │   场景标签: [村庄]           │  [物品][装备]   │
│ 升级行动     │                             │                 │
│ 战斗属性     │   循环行动                   │  资源 / 矿物    │
│             │   ┌─────────────────────┐   │  装备 / 动物材料 │
│             │   │ 事件卡片             │   │                 │
│             │   │ 名称/描述/标签/按钮  │   │  [图标]名称×数量 │
│             │   └─────────────────────┘   │                 │
│             │                             │                 │
└─────────────┴─────────────────────────────┴─────────────────┘
```

## 设计原则

1. **信息密度优先**：放置游戏的核心乐趣来自数字增长，三个面板同时展示属性、事件、库存，减少切换成本。
2. **实时反馈**：进度条、数值变化、浮动提示等即时响应服务端增量更新。
3. **操作路径最短**：高频操作（启停循环、加入队列、切换场景）均在当前界面直接完成。
4. **数值可读性**：大数量使用 `k`/`M` 等缩写（如 `118k`），避免数字过长。

## 状态同步机制

前端通过 WebSocket 与后端保持长连接：

- **完整状态**：连接建立时接收一次 `state.full` 完整快照，初始化所有面板。
- **增量更新**：服务端 Tick 推进过程中有变更时主动推送 `state.diff`，前端局部刷新对应区域。
- **断线重连**：WebSocket 断开时前端自动重连（指数退避）。重连后复用已有 Session（若仍在 Grace 期内）或创建新 Session，接收 `state.full` 完整快照平滑恢复。
- **无应用层心跳**：当前实现没有应用层 `heartbeat` 消息。服务端通过 WebSocket 层的 `onclose` 回调检测断开，触发 Grace Timer（默认 30s）。底层传输层会按 30s 间隔发送 WebSocket ping 帧保活。
- **优雅关闭**：用户 logout 或离开 `/main` 页面时，前端调用 `disconnect()` 设置 `intentionalClose=true`，阻止自动重连。
