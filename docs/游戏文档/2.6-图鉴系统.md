# 图鉴系统

图鉴系统（Bestiary）记录玩家已解锁的事件、已发现的物品和已访问的区域。它影响前端的事件可见性判断（未解锁的内容不展示），并通过 `state.full` / `state.diff` 同步给客户端。

## 数据结构

```go
// backend/internal/bestiary/state.go
type State struct {
    userID int64
    events map[gameconfig.EventID]bool   // 已解锁的事件
    items  map[item.Item]bool             // 已发现的物品（含 State 变体）
    areas  map[gameconfig.MapID]bool     // 已访问的区域（纯内存，不入库）

    dirtyEvents map[gameconfig.EventID]bool   // 本周期新解锁的事件（待刷盘）
    dirtyItems  map[item.Item]bool             // 本周期新发现的物品（待刷盘）

    recorder *record.Recorder
}
```

| 类别 | 存储位置 | 触发解锁 | DB 持久化 |
|------|----------|----------|-----------|
| 事件 | `events` | `UnlockEvent(id)` | `player_unlocked_events` |
| 物品 | `items` | `UnlockItem(it)` | `player_discovered_items` |
| 区域 | `areas` | `UnlockArea(id)` | 无（运行时推断） |

## 解锁机制

### 事件解锁

```go
func (s *State) UnlockEvent(id gameconfig.EventID) {
    if s.events[id] { return }
    s.events[id] = true
    s.dirtyEvents[id] = true
    s.record(eventUnlock(id))
}
```

**触发时机**：
- 执行 `upgrade` 类型事件时，事件处理器调用 `ctx.UnlockEvent(eventID)`。
- 执行 `instant` 类型事件时（若设计为一次性解锁剧情）。

### 物品发现

```go
func (s *State) UnlockItem(it item.Item) {
    if s.items[it] { return }
    s.items[it] = true
    s.dirtyItems[it] = true
    s.record(itemUnlock(it))
}
```

**触发时机**：
- `PlayerSession.AddItem(it, qty)` 在 `qty > 0` 时调用 `s.best.UnlockItem(it)`。
- `PlayerSession.Equip` 替换旧装备时，旧装备返回库存 → `UnlockItem(old)`。
- `PlayerSession.Unequip` 卸下装备返回库存 → `UnlockItem(it)`。

> 注意：`items` 的 key 是完整的 `item.Item{ID, State}`，但数据库只存储 `item_id`（不含 `item_state`）。这意味着不同 State 的同一物品在图鉴中只记录一次发现。

### 区域解锁

```go
func (s *State) UnlockArea(id gameconfig.MapID) {
    if s.areas[id] { return }
    s.areas[id] = true
    s.record(areaUnlock(id))
}
```

**触发时机**：当前未实现自动触发，区域图鉴为预留字段。

## 数据库持久化

### 表结构

```sql
CREATE TABLE player_unlocked_events (
    user_id     INTEGER  NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    event_id    INTEGER  NOT NULL,
    unlocked_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, event_id)
);

CREATE TABLE player_discovered_items (
    user_id       INTEGER  NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    item_id       INTEGER  NOT NULL,
    discovered_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, item_id)
);
```

### 加载

`CreateSession` 从 DB 加载：

```go
eventRows, _ := q.LoadUnlockedEvents(ctx, userID)
best.LoadEvents(ids)

itemRows, _ := q.LoadDiscoveredItems(ctx, userID)
best.LoadDiscoveredItems(itemRows)
```

`LoadDiscoveredItems` 从 DB 行重建 `item.Item{ID: item.ID(r.ItemID), State: 0}`。

### 刷盘

```go
func (s *State) Flush(ctx context.Context, q *dbgen.Queries) error {
    for id := range s.dirtyEvents {
        q.UpsertUnlockedEvent(ctx, UserID: s.userID, EventID: int64(id))
    }
    s.dirtyEvents = make(map[gameconfig.EventID]bool)

    for it := range s.dirtyItems {
        q.UpsertDiscoveredItem(ctx, UserID: s.userID, ItemID: int64(it.ID))
    }
    s.dirtyItems = make(map[item.Item]bool)
    return nil
}
```

## 序列化与同步

### Diff 序列化（增量更新）

`bucket.go` 中的 `Bucket` 实现 `record.RecordBucket`：

```go
type Bucket struct {
    entries map[string]unlockEntry   // key = type + "/" + id
}

type unlockEntry struct {
    typ string   // "event" / "item" / "area"
    id  string   // 字符串 ID（前端展示用）
}
```

**合并规则**：同一 `type + id` 去重，仅保留一条。

**序列化示例**：

```json
{
  "bestiary": [
    {"type": "event", "id": "felling_oak_tree"},
    {"type": "item", "id": "oak_logs"}
  ]
}
```

### Full 序列化（完整快照）

`provider.go` 中的 `SerializeFull`：

```go
func (p *provider) SerializeFull(state any) (proto.Message, error) {
    st := state.(*State)
    var out []*pb.BestiaryFull
    for eid := range st.events {
        out = append(out, &pb.BestiaryFull{Type: "event", Id: eid.String()})
    }
    for it := range st.items {
        out = append(out, &pb.BestiaryFull{Type: "item", Id: itemBestiaryID(it)})
    }
    for mid := range st.areas {
        out = append(out, &pb.BestiaryFull{Type: "area", Id: mid.String()})
    }
    sort.Slice(out, ...)
    return &pb.StateFull{Bestiary: out}, nil
}
```

### ID 映射规则

图鉴系统在前端使用**字符串 ID**，后端内部使用**数字 ID**。

| 类别 | 内部类型 | Wire (protobuf) |
|------|----------|-----------------|
| 事件 | `gameconfig.EventID` (int64) | `eid.String()` → 查注册表得 `"felling_oak_tree"` |
| 区域 | `gameconfig.MapID` (int64) | `mid.String()` → 查注册表得 `"village"` |
| 物品 | `item.ID` (int32) | `itemBestiaryID(it)` → 优先用 `ItemDef.StringID()`，fallback 到 `it.String()` |

物品 ID 的处理最为关键。`itemBestiaryID` 优先查找 `gameconfig` 中注册的物品定义：

```go
func itemBestiaryID(it item.Item) string {
    if def, ok := gameconfig.GetItemDefByID(it.ID); ok {
        return def.StringID()   // 如 "oak_logs"
    }
    return it.String()           // fallback: "<item-19>/0"
}
```

**必须保证**所有在奖励中发放的物品都在 `actions.json` 中有定义，否则前端会收到无法识别的 ID。

## 前端处理

前端 `game.ts` 的 `applyStateFull` 和 `applyStateDiff` 处理图鉴数据：

```ts
// state.full 初始化
state.unlocked_events = []
state.seen_items = []
for (const b of full.bestiary ?? []) {
  if (b.type === 'event') state.unlocked_events.push(b.id)
  else if (b.type === 'item') state.seen_items.push(b.id)
}

// state.diff 增量合并
if (diff.bestiary) {
  const existingEvents = new Set(state.unlocked_events)
  const existingItems = new Set(state.seen_items)
  for (const b of diff.bestiary) {
    if (b.type === 'event') existingEvents.add(b.id)
    else if (b.type === 'item') existingItems.add(b.id)
  }
  state.unlocked_events = Array.from(existingEvents)
  state.seen_items = Array.from(existingItems)
}
```

`seen_items` 用于判断 crafting 事件的可见性（ requirements 中的 `item` 类型条件）。

## 可见性规则

前端展示事件时，需满足以下条件：

1. `requirements` 中所有 `type: "event"` 的条目对应的 `id` 必须在 `unlocked_events` 中。
2. `requirements` 中所有 `type: "item"` 或 `type: "fluid"` 的条目对应的 `id` 必须在 `seen_items` 中。
3. 技能等级门槛通过 `state.skills` 判断。

满足全部条件的事件才会在前端展示。
