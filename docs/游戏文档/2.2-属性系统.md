# 属性系统

属性系统负责管理角色、装备、工具等实体的能力数值。核心分为三层：**属性定义**、**属性注册表**、**属性修饰**。

## 整体架构

```
┌─────────────────┐
│  属性定义层      │  attributes.json 定义有哪些属性、类型、默认值
├─────────────────┤
│  属性注册表      │  全局只读注册表，启动时加载，维护依赖图和反向依赖图
├─────────────────┤
│  修饰器系统      │  描述"如何改变属性值"
├─────────────────┤
│  属性实例        │  每个 PlayerSession 一个，内存中懒计算
└─────────────────┘
```

## 1. 属性定义

`attributes.json` 定义属性的元数据：

| 字段 | 说明 |
|------|------|
| `id` | 字符串标识，如 `"physical_power"`，仅用于配置和前端展示 |
| `name` | 显示名称 |
| `default_value` | 默认值（浮点数）|
| `min_value` | 最小值限制（可选）|
| `max_value` | 最大值限制（可选）|
| `direction` | `positive`（越大越好）/ `negative`（越小越好），影响 UI 颜色 |
| `group` | 属性分组，如 `"combat"`、`"production"`、`"base"`，用于 UI 分组 |
| `desc` | 描述文本 |

运行时，每个玩家的属性实例额外维护：

| 字段 | 说明 |
|------|------|
| `cached_value` | 上次计算出的最终值 |
| `dirty` | 布尔标记，`true` 表示缓存已失效，下次读取需重算 |

## 2. 属性注册表

全局不可变注册表，启动时从 `attributes.json` + `attr_registry.json` 加载：

- **双向映射**：`byID`、`byString`、`idToString`
- **依赖图**：`forward`（一个属性的修饰器引用了哪些其他属性）和 `reverse`（哪些属性的修饰器引用了我）
- **循环依赖检测**：加载时 DFS 检测，发现环则启动失败

## 3. 修饰器系统

修饰器（Modifier）是改变属性值的最小单元。

```go
type Modifier struct {
    AttrID   AttributeID   // 目标属性数字 ID
    Op       OpType        // ADD / MULTIPLY / OVERRIDE
    Value    float64       // 固定数值（与 RefAttr 二选一）
    RefAttr  AttributeID   // 引用的属性数字 ID（与 Value 二选一）
    Display  DisplayMode   // FIXED / PERCENT / PER_SECOND，仅影响前端展示
    Source   string        // 来源标识，如 "equipment:sword_001"
    Priority int           // 计算优先级
}
```

### 操作类型

| 枚举值 | 说明 | 计算阶段 |
|--------|------|----------|
| `ADD` | 加法修饰，累加到当前值 | 第一阶段 |
| `MULTIPLY` | 乘法修饰，效果为 `×(1 + value)` | 第二阶段 |
| `OVERRIDE` | 覆盖修饰，直接替换最终值 | 第三阶段 |

### 修饰器来源

| 来源 | 触发时机 | 操作 |
|------|----------|------|
| `equipment` | 穿戴装备 | `AddModifiers("equipment:"+id, mods)` |
| `tool` | 装备/卸下工具 | 同 equipment |
| `skill` | 技能等级变化 | `UpdateModifiers("skill:"+id, newMods)` |
| `base` | 角色创建 | `AddModifiers("base", baseMods)` |

### 生命周期管理

```go
func (inst *Instance) AddModifiers(source string, mods []Modifier)
func (inst *Instance) RemoveModifiers(source string)
func (inst *Instance) UpdateModifiers(source string, mods []Modifier)
```

- `source` 作为分组键，同一来源的修饰器**原子替换**。
- 变更后自动执行脏标记传播：被变更修饰器指向的目标属性 + 反向依赖图中所有引用它的属性全部标记为 dirty。
- 同时向当前 `record.Recorder` 写入 `AttributeDirtyRecord`。

## 4. 属性实例

每个 `PlayerSession` 拥有一个独立的 `attribute.Instance`，随连接创建/销毁。

**连接建立时重建**：
1. 挂载静态修饰器（来自 `attributes.json` 的 `modifiers` 段）
2. 遍历穿戴装备 → `AddModifiers("equipment:"+id, mods)`
3. 遍历装备工具 → `AddModifiers("tool:"+id, mods)`
4. 遍历技能等级 → `UpdateModifiers("skill:"+id, mods)`
5. 全部标记为 dirty（首次读取时懒计算）

**连接期间**：写操作直接修改该实例，不重建。读场景直接读取 `cached_value`。

## 5. 计算流程

懒计算 + 三阶段计算 + 缓存：

```go
func (inst *Instance) GetFinal(attrID AttributeID) float64 {
    // 缓存命中且未脏，直接返回
    if !inst.dirty[attrID] {
        if v, ok := inst.cached[attrID]; ok { return v }
    }

    // 默认值
    value := registry[attrID].default_value
    mods := inst.byAttr[attrID]   // 已按 priority 排序

    // Phase 1: ADD（含 RefAttr 递归解析）
    for _, m := range filter(mods, ADD) {
        if m.RefAttr != 0 {
            value += inst.GetFinal(m.RefAttr)   // 递归
        } else {
            value += m.Value
        }
    }

    // Phase 2: MULTIPLY
    for _, m := range filter(mods, MULTIPLY) {
        if m.RefAttr != 0 {
            value *= (1 + inst.GetFinal(m.RefAttr))
        } else {
            value *= (1 + m.Value)
        }
    }

    // Phase 3: OVERRIDE（最后一个生效）
    for _, m := range filter(mods, OVERRIDE) {
        value = m.Value
    }

    // 截断到 [min_value, max_value]
    // 写缓存，清脏标记
    inst.cached[attrID] = value
    inst.dirty[attrID] = false
    return value
}
```

### 脏标记传播

```go
func (inst *Instance) markDirty(attrID AttributeID) {
    if inst.dirty[attrID] { return }
    inst.dirty[attrID] = true
    // 向 recorder 写入 dirty 记录
    if inst.recorder != nil {
        inst.recorder.Bucket("attribute").(*Bucket).MarkDirty(attrID)
    }
    // 递归标记所有反向依赖
    for _, depID := range inst.reg.ReverseDeps(attrID) {
        inst.markDirty(depID)
    }
}
```

## 6. 临时计算上下文

支持通过 `Context` 传入临时参数用于单次计算（如事件结算、战斗模拟、预览强化结果）：

```go
type Context struct {
    tempBase map[AttributeID]float64   // 临时覆盖默认值
    tempMods []Modifier                // 本次计算额外生效的临时修饰器
}
```

- 临时上下文**不参与缓存**，每次调用独立计算。
- 临时上下文**不触发脏标记**。

典型场景：事件结算时某 Buff 临时提供一次性产出加成。

## 7. 更新记录与序列化

### Diff 记录

属性系统内部只记录 dirty 标记，不记录前后值：

```go
type Bucket struct {
    dirty map[AttributeID]bool
}
```

**合并规则**：同一 `AttrID` 在同一命名空间内去重，仅保留一条。

### Diff 序列化

`SerializeDiff` 时**现场计算最终值**，并收集该属性上的完整修饰器状态一并推送：

```json
{
  "attribute": [
    {
      "attr_id": "final_attack_power",
      "final_value": 33.0
    }
  ]
}
```

### Full 序列化

`SerializeFull` 输出所有注册属性及其最终值 + 修饰器列表。

## 8. 持久化边界

属性系统**不直接依赖数据库**。数据库只存"状态事实"：

| 数据 | 数据库表 | 说明 |
|------|----------|------|
| 穿戴的装备 | `player_equipment` | 槽位 → 物品映射 |
| 技能等级 | `player_skills` | 决定技能类修饰器数值 |

属性实例常驻于 `PlayerSession` 内存中，连接断开后销毁，下次连接重新从状态事实构建。
