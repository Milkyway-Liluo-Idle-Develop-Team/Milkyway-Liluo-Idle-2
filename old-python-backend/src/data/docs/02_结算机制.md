# 结算机制

## 离线结算原理

银河莉萝放置采用**离线回溯结算**模式。服务器不运行实时游戏循环，而是在客户端发起同步请求时，根据上次同步以来经过的时间，回放所有队列中的行动。

```
elapsed = 当前时间 - 上次同步时间
```

这意味着：
- 玩家离线时，进度照常积累
- 每次客户端连接/同步时，服务器一次性计算所有离线期间的产出
- 结算结果完全确定（基于队列和时间差）

## 结算流程

### 1. 计算经过时间
```
remaining = elapsed = now - last_sync_time
```

### 2. 回放队列
服务器从队列的当前位置（`queue_index`）和当前进度（`queue_progress_seconds`）开始，按顺序处理行动：

```
while remaining > 0 且 queue_index < 队列长度:
    取出当前行动
    if 行动需求不满足: 停止结算
    
    根据行动类型处理:
      loop  → 循环执行直到时间或材料耗尽
      instant → 立即执行一次，移动到下一个
      upgrade → 解锁一次，移动到下一个
```

### 3. 循环行动的详细结算

对于 `loop` 类型的行动，结算流程如下：

```
计算实际循环时间 loop_time（受属性修饰）
可执行次数 = floor((已积累进度 + 剩余时间) / loop_time)

for 每一次执行:
    if 需求不满足: 跳出
    扣除材料消耗
    发放奖励（受属性修饰 + 概率取整）
    增加经验（受属性修饰）

更新已消耗时间
更新剩余进度 = 已积累进度 % loop_time
```

### 4. 结算日志

每次结算会生成一份结算日志（`settlement_log`），记录每个行动执行了多少次、获得了多少经验，供前端展示。

格式示例：
```json
[
    {"event_id": "felling_oak_tree", "iterations": 150, "experience": 3000},
    {"event_id": "making_oak_plank", "iterations": 50, "experience": 1000}
]
```

## 队列机制

### 设置队列
玩家可以随时设置新的行动队列。设置新队列时：
1. 先结算旧队列的所有进度
2. 验证所有行动 ID 是否有效
3. 重置队列索引和进度为 0
4. 更新同步时间

### 队列状态
- `queue`：行动 ID 列表，如 `["felling_oak_tree", "making_oak_plank"]`
- `queue_index`：当前正在执行的行动索引
- `queue_progress_seconds`：当前行动已累积的时间进度

### 队列推进
- 循环行动：在时间耗尽前持续执行，不会自动推进到下一个
- 即时/升级行动：执行一次后自动推进到队列中的下一个行动

## 需求检查

每个行动可以设置多项前置需求，结算时会逐一检查：

| 需求类型 | 说明 | 检查方式 |
|---------|------|---------|
| `skill` | 技能等级要求 | 检查玩家对应技能的当前等级 |
| `item` | 物品数量要求 | 检查背包中的物品数量 |
| `event` | 升级行动解锁要求 | 检查是否已解锁对应的升级行动 |
| `fluid` | 流体数量要求 | 检查流体资源数量 |

### 比较方式

需求可以指定比较方式（`comparison_types`）：

| 比较类型 | 含义 |
|---------|------|
| `bigger_or_equal`（默认） | >= |
| `bigger` | > |
| `equal` | == |
| `smaller` | < |
| `smaller_or_equal` | <= |

### 材料消耗规则

**重要**：只有 `item` 和 `fluid` 类型、且**没有指定 `comparison_types`** 的需求才会被扣除为消耗材料。

例如：
```json
{"type": "item", "id": "oak_logs", "value": 1}
```
→ 每次执行扣除 1 个橡木原木（消耗）

```json
{"type": "skill", "id": "crafting", "comparison_types": "bigger_or_equal", "value": 20}
```
→ 只检查制造技能是否 >= 20，不扣除（前置条件）

```json
{"type": "item", "id": "wooden_sword", "comparison_types": "smaller_or_equal", "value": 0}
```
→ 检查木剑数量 <= 0（即没有木剑），不扣除
