# 装备赋能系统

## 装备强化

### 强化成功率计算

装备强化是物品提升最直接的手段，每一个可以强化的物品的强化都由以下信息控制:

- max_upgrade -> 最大强化
- upgrade_curve: 线性强化曲线

强化曲线由以下节点构成一个分段线性函数: (除了 requirement 都是数字, 数值线性变化的, 和大多数循环行动的物品不同, requirements 的物品数量需求是向下取整的)

- level -> 节点强化等级,
- recommend_level -> 节点推荐赋能等级,
- basic_success_rate -> 节点基础成功率,
- ability_multiplier -> 装备能力倍增
- requirements? -> 和循环行动一样的物品需求, 空则是没有物品需求

> 注意, 强化等级为 $x$ 的数据指的是从强化等级 $x$ 升级到强化等级 $x+1$

其中, 装备实际成功率和基础成功率和当前玩家的装备、赋能等级和推荐等级的关系是:

若玩家的赋能等级 小于 推荐等级时:

$$ 成功率 = 基础成功率 * 0.99 ^ {推荐等级 - 玩家赋能等级}$$

否则:

$$ 成功率 = 基础成功率 * ( 1 + (\frac{玩家赋能等级 - 推荐等级}{ 35 } ) ^2 ) $$

此外 成功率还受到玩家 赋能成功率倍增影响:

$$ 最终成功率 = 成功率 * ( 1 + 玩家强化成功率倍增 ) $$


> 例子: 如果一件装备有节点:
{
    "level": 5,
    "recommend_level": 15,
    "basic_success_rate": 0.5,
    "ability_multiplier": 2,
    "requirements": [
        {
            "id": A,
            "num": 5
        }
    ],
},{
    "level": 10,
    "recommend_level": 20,
    "basic_success_rate": 0.4,
    "ability_multiplier": 3,
    "requirements": [
        {
            "id": A,
            "num": 10
        },{
            "id: B,
            "num": 1,
        }
    ],
}
则它在强化等级为 8 想升到 9 时插值 level = 8, 有推荐等级18, 成功率 0.44, 装备能力倍增: 2.6, 强化需求: A 为 8 个, B为 0 个(向下取整)

### 强化过程和结果

强化除了基础的材料需求以外, 一般都需要这个装备的一个原始版本进行 "保护" (即没有任何强化、附魔等的白板物品). 和大部分放置游戏不同, 本游戏中的保护是强制的, 且暂时没有设置保护石等万能物品.

装备强化所需要的时间为固定 $5\mathrm{s}$, 这是为了给用户充足的反应时间, 未来也不会加入修改这个数值的装备和工具等, 进行强化给予的经验固定为 $50 * 1.005 ^ {强化推荐等级}$.

装备强化实际成功所需次数满足期望为 $1 / 成功率$ 的泊松分布, 以减少正常强化的方差, 我们会记录一个装备的单次强化的失败次数, 考虑到用户在给同一个装备升级中间经过了很长的时间, 即多次成功率不一样, 因此我们不会预先指定一个装备的成功次数, 而是将单次成功率变为 (这个成功率是后端计算, 前端现实的还是基础的成功率):

$$ P(k\rightarrow k+1 | \lambda = \frac{1}{possibility}) = \frac{\lambda^k / k!}{\sum_{l=k}^{+\infty}\frac{\lambda^l}{l!}}$$

装备有两套提供的属性值, 一套是不受强化加成的基础属性, 一套是受 "ability_multiplier" 影响的可倍增的属性:

装备最终属性 = 基础属性 + 可倍增属性 * 强化带来的装备能力倍增

## 装备分解

和银河奶牛放置类似, 升级到 +20 的装备可以进行分解, 产出对应材质种类的精华. 例如+20的木剑可以分解产生 10 个木制精华.

木之精华的作用是: 装备从 +20 升级到 +30 的每一级都需要精华:

| 等级 | 需要的精华数量 |
| ----- | ----- |
| 20 | 10 |
| 21 | 10 |
| 22 | 20 |
| 23 | 30 |
| 24 | 40 |
| 25 | 60 |
| 26 | 80 |
| 27 | 100 |
| 28 | 130 |
| 29 | 160 |

> 如果后面有需要, 可以开放所有等级的装备的分解, 暂时想不到能分解成什么东西.

## 装备锻造

每一个装备和工具都有一个 "forge_slot" 字段, 表示了一个装备和工具最多可以锻造的槽位. 每一个装备可以花费对应精华来给一个有空槽位的装备提供对应的能力.

目前打算弄的几个能力有:

- 木制能力:
- - 当给予在任意工具时: 给予伐木产量 +10% 的加算 buff. 如果装备有耐久值机制, 则会以 1分钟 6 点的速度回复耐久值.
- - 当给予在任意装备时: 会提供 +10% 的回复速度 buff. 如果装备有耐久值机制, 则会以 1分钟 6 点的速度回复耐久值.

- 石制能力:
- - 当给予在任意工具时: 给予挖矿产量 +15% 的加算 buff. 
- - 当给予在任意装备时: 会提供 +15% 的防御乘算buff.

- 铜制能力:
- - 当给予在任意工具时: 没想好.
- - 当给予在任意攻击武器时: 攻击会给予持续时间无限、不可叠加的 20% 易伤
- - 当给予在任意非攻击武器时: 没想好

- 铁制能力:
- - 当给予在任意工具时: 如果此工具的效果生效, 那么将会给予随时间提升的装备效果, 不使用时清零. 上限是 2小时 提供 50% 的装备效果倍增
- - 当给予在任意装备时: 同上, 但是上限改为 2小时 提供 25%

## 装备附魔

这一块暂时没做, 设想是和 MC 一样使用附魔书弄, 然后每个装备附魔上, 两个同等级的附魔合成更高等级的附魔.

> 暂定可能存在的附魔有?
锋利、时运、效率、耐久(铜制装备用) 等

## 悄悄话: 各个装备的定位是怎么样的?

木制装备 -> 石制装备 -> 铁制装备 -> 钻石装备(以后) -> 合金装备都是很经典的 MC 的基础升级.

当然我们不能否认这个设计不好, 我们只需要做几个有特色的装备合集就行了.

铜装备: 我采纳了社区觉得 mc 觉得铜装备不够有特色, 提出了一个观点: 铜装备在耐久高的时候效率高, 耐久低的时候效率低.

金装备: 借鉴 QB 放置的, 金装备提供额外的锻造槽位

青金石 / 或别的矿物装备? 提供额外的附魔槽位

### 如何避免后期装备性能踩头?

这一点是不能避免的, 我们想让所有装备都成为一个合适的过渡. 因此我把装备强化的效果拉的很大, 比如 30 级提供 13 倍的装备性能.

这一点还不能消除踩头的问题, 所以借鉴匠魂弄了个锻造系统.
