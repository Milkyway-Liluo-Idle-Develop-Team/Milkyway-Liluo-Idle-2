# 数值公式

## 经验与升级

### 经验获取

每次执行行动获得的经验值由行动定义中的 `experience` 字段决定，并受以下属性修饰：

```
实际经验 = 基础经验 × (1 + exp_gain_multiplier)
```

例如：砍伐橡木基础经验 20，装备克苏鲁木雕（exp_gain_multiplier = 0.1）后：
```
实际经验 = 20 × (1 + 0.1) = 22
```

### 升级机制

每个技能的经验值是**累积总经验**。系统维护一张等级-累积经验对照表，当前等级由总经验在表中的位置决定：

```
当前等级 = 二分查找(总经验在经验表中的位置) + 1
```

前几级的累积经验需求：

| 等级 | 累积经验需求 |
|------|------------|
| 1 → 2 | 80 |
| 2 → 3 | 300 |
| 3 → 4 | 699 |
| 4 → 5 | 1,304 |
| 5 → 6 | 2,140 |
| 10 → 11 | 13,159 |
| 20 → 21 | 62,173 |
| 50 → 51 | 529,215 |
| 100 → 101 | 4,216,121 |

等级上限约为 1000 级，后期升级所需经验呈指数级增长。

## 生产速度

### 基础 loop_time

每个循环行动有一个基础 `loop_time`（秒），表示执行一次所需的时间。

### 属性修饰后的 loop_time

如果行动关联了某项技能（`need_skill`），且玩家拥有该技能的生产倍率属性，则实际循环时间缩短：

```
实际循环时间 = 基础循环时间 / (1 + {skill}_speed_multiplier)
```

例如：砍伐橡木 loop_time = 2秒，装备木斧（felling_speed_multiplier = 0.1）：
```
实际循环时间 = 2 / (1 + 0.1) = 1.818 秒
```

装备石斧（felling_speed_multiplier = 0.2）：
```
实际循环时间 = 2 / (1 + 0.2) = 1.667 秒
```

### 基础奖励

所有行动的奖励由 `rewards` 字段控制, 表示循环一次的奖励.

### 属性修饰后的 rewards

如果循环行动关联的对应技能玩家拥有相应倍率, 则每次行动的奖励将会提高.

- 这也是不设置装备向上合成的原因之一, 可能 1 个木镐向上升级为 2 个石镐, 向上升级为 4 个铜镐

```
实际奖励 = 基础奖励 * ( 1 + {skill}_production_multiplier )
```

### 概率取整

属性修饰后的奖励数量可能出现小数。系统采用**概率取整**机制处理：

```
对于数值 N.f（整数部分 N，小数部分 f）：
  - f 概率向上取整为 N+1
  - (1-f) 概率向下取整为 N
```

**示例**：
- 3.7 → 70% 概率得到 4，30% 概率得到 3
- 4.5 → 50% 概率得到 5，50% 概率得到 4
- 10.0 → 必定得到 10（无小数部分）

这保证了长期期望值与精确计算一致，同时避免了"永远少给"或"永远多给"的偏差。

## 战斗公式

### 伤害计算

游戏中的伤害 (目前) 分为两种, 一种是物理伤害, 另一种是奥术伤害.

```
进攻方基础伤害 = 原始伤害 * ( 1 + 基础伤害倍率 )
```

- 这里的最终基础伤害来自于 power_multiplier, 和 damage_multiplier 不同.

```
造成的物理原始伤害 = ( 0.9 + random() * 0.2 ) * ( 物理伤害 ^ 2) / ( 物理伤害 + 物理防御)  * ( 1 +攻击方最终伤害倍率加成 ) / ( 1 + 防御方最终伤害减免倍率加成 )
```

换句话说, 造成的伤害 (如果没计算上倍率) 和防御值是一个平移的反比例函数, 且满足防御值为 0 时伤害和防御的导数为 -1.

奥术原始伤害公式如下:

```
造成的奥术原始伤害 =  ( 0.9 + random() * 0.2 ) * 奥术伤害 / ( 1 + 奥术抵抗 ) * ( 1 +攻击方最终伤害倍率加成 ) / ( 1 + 防御方最终伤害减免倍率加成 )
```

计算完原始伤害后, 进入格挡和暴击计算: 格挡概率由以下公式决定, 而且奥术伤害不受格挡值暴击计算.

```
格挡概率 = 1 - 100 / ( 100 + 格挡值 ) / ( 1 + 格挡率倍率加成 )

```

格挡后, 伤害会有一定减少:

同时也会进行暴击计算:

```
暴击概率 = clamp( 暴击率, 0, 1)
最终伤害 = round( 造成的原始伤害 * 暴击倍率(若暴击, 否则为1) / 格挡倍率(若格挡, 否则为 1) )
```

```
暴击伤害倍率
```
在进行完之后进行闪避计算, 当闪避时收到的伤害为 0, 即 miss:

```
闪避率 = clamp( 1 / (1 + ( clamp ( 精准 / 闪避率, 0, 10) ) ^ 2), 0, 1) * ( 1 + 防御方闪避率倍率加成 ) / (1 + 攻击方精准倍率加成)
```



## 装备升级曲线（预览）

每件可升级装备（`upgradable: true`）拥有一条升级曲线（`upgrade_curve`），定义了不同强化等级段的参数：

| 参数 | 说明 |
|------|------|
| level | 强化等级起始点 |
| recommend_level | 建议角色等级 |
| basic_success_rate | 基础成功率 |
| ability_multiplier | 能力倍率 |

典型木制装备升级曲线：

| 等级段 | 成功率 | 倍率 |
|-------|--------|------|
| 0-4 | 52% | 1.0x |
| 5-9 | 37% | 1.5x |
| 10-14 | 28% | 2.5x |
| 15-19 | 20% | 4.0x |
| 20-24 | 15% | 6.0x |
| 25-29 | 12.5% | 9.0x |
| 30 | 10% | 13.0x |

*注：装备强化系统的完整实现尚在开发中。*
